# Контекстно-свободный поиск путей в графах (CFPQ)

**Тема:** Обнаружение структурных циклов в направленных помеченных графах методами линейной алгебры

---

## 1. Цели: 
В рамках работы сформулированы и решены 2 исследовательские задачи:

1. Обнаружение цикла типа **a^n** - вершина достигает себя по цепочке рёбер одного типа.
2. Обнаружение цикла типа **a^m b^k a^i b^j** - замкнутый контур из чередующихся рёбер двух типов с произвольными длинами блоков

---

## 2. Базовый алгоритм CFPQ

Алгоритм принимает на вход направленный граф `G = ⟨V, E, L⟩` и КС-грамматику. Граф представляется набором разреженных булевых матриц смежности - по одной матрице на каждую метку ребра. Для каждого нетерминала `A` грамматики создаётся матрица `T_A` размера `|V| × |V|`.

Алгоритм выполняет итеративное обновление по правилам грамматики:

- **Инициализация:** для каждого ребра `(i, x, j)` и правила `A -> x` добавляем `A` в `T_A[i, j]`.
- **Итерация:** для каждого правила `A -> B C` вычисляем `T_A |= T_B × T_C`.
- **Остановка:** когда ни одна матрица `T_A` не изменилась (достижение неподвижной точки).

Теоретический результат: элемент `T_S[i, j] = 1` <=> существует путь из вершины `i` в вершину `j`, слово которого порождается стартовым нетерминалом `S` грамматики

---

## 3. Исследование 1: обнаружение цикла a^n

### 3.1. Формулировка задачи

Первое исследование отвечает на вопрос: существует ли в графе вершина, достигающая сама себя по цепочке рёбер определённого типа?

В терминах наследования это означает: является ли класс A прямым или транзитивным наследником самого себя - что запрещено в любой системе типов.

### 3.2. Алгоритм обнаружения

Алгорит реализован в файле research/check_diagonal.py:

1 Запустить `cfpq_matrix()` с грамматикой транзитивного замыкания. Получить словарь результирующих пар `R_S`.
2 Извлечь диагональ: отобрать все пары `(i, j) ∈ R_S`, где `i = j`.
3 Для каждой найденной цикличной вершины запустить BFS по рёбрам графа и восстановить конкретный путь, образующий цикл.

Использованная грамматика (`grammars/subclassof_cnf.txt`) задаёт транзитивное замыкание по рёбрам `subclassOf`:

```
S -> subclassOf S
S -> subclassOf
```

**проверка диагонали** - это операция O(|V|) после уже выполненного CFPQ. Весь вычислительный бюджет уходит на построение матрицы `T_S`, а не на поиск нарушений в ней.

### 3.3. Тестовые данные

Подготовлены 2 графа:

**`graphs/inheritance_cycle.csv`** - 4 вершины, 4 ребра. Содержит цикл `0 -> 1 -> 2 _> 0` по `subclassOf`, плюс вершина `3 -> 1`:

```
0,subclassOf,1
1,subclassOf,2
2,subclassOf,0
3,subclassOf,1
```

**`graphs/inheritance_ok.csv`** - 4 вершины, 3 ребра. Линейная иерархия без цикла:

```
0,subclassOf,1
1,subclassOf,2
3,subclassOf,1
```

### 3.4. Результаты

#### Граф с циклом (`inheritance_cycle.csv`)

CFPQ выполнен за **2 итерации**, найдено **12 пар** в `R_S`.

| Пара (i, j) | Тип |
|-------------|-----|
| **(0, 0)** | - **ЦИКЛ**: вершина 0 достигает себя |
| (0, 1) | нормальное достижение |

| (1, 0) | нормальное достижение |

```
R_S = {(0,1), (0,2), (1,2), (3,1), (3,2)}
```

### 3.5. Выводы по исследованию 1

Метод работает корректно и позволяет точно локализовать **все** вершины, участвующие в запрещённом цикле. Показывается точная цепочка классов, которую нужно разорвать, вычислительная стоимость проверки диагонали пренебрежимо мала по сравнению с самим CFPQ

## 4. Исследование 2: обнаружение цикла a^m b^k a^i b^j

### 4.1. Формулировка задачи

Второе исследование рассматривает паттерн: цикл, при котором чередуются 2 типа рёбер, формально разыскивается четвёрка вершин `(i0, i1, i2, i3)` такая, что:

```
i0 ->(a^m)-> i1  -  цепочка из m рёбер типа a
i1 ->(b^k)-> i2  -  цепочка из k рёбер типа b
i2 ->(a^i)-> i3  -  цепочка из i рёбер типа a
i3 ->(b^j)-> i0  -  цепочка из j рёбер типа b, замыкающих цикл
```

Числа `m, k, i, j` не заданы заранее - это произвольные натуральные числа. Нельзя перебирать все возможные значения. Нужен алгоритм, который находит такой цикл для любых `m, k, i, j` одновременно.

В контексте объектных моделей этот паттерн означает: класс A наследует через цепочку некий класс B, который содержит как поле класс C, от которого через наследование и включение замыкается обратно в A - образуя взаимную рекурсивную зависимость.

### 4.2. Пофазный алгоритм

Ключевая идея: вместо 1 большой грамматики запускаются 4 последовательных вычисления транзитивного замыкания. На каждой фазе множество рассматриваемых вершин-источников сужается

| Фаза | Вычисление | Источники | Результат |
|------|------------|-----------|-----------|
| 1 | `T_a  = transitive_closure(a-рёбра, все вершины)` | все `n` вершин | множество a-достижимостей |
| 2 | `T_b1 = transitive_closure(b-рёбра, a-targets)` | столбцы `T_a` | b-достижимости от границы a->b |
| 3 | `T_a2 = transitive_closure(a-рёбра, b1-targets)` | столбцы `T_b1` | второй блок a |
| 4 | `T_b2 = transitive_closure(b-рёбра, a2-targets)` | столбцы `T_a2` | второй блок b |
| 5 | `combined = T_a × T_b1 × T_a2 × T_b2` | - | диагональ - цикличные вершины |

Финальная проверка сводится к **диагональному тесту** - той же идее, что и в Исследовании 1, но применённой к произведению четырёх матриц: если `combined[i0, i0] = 1`, то вершина `i0` участвует в цикле `a^m b^k a^i b^j`.

Транзитивное замыкание на каждой фазе вычисляется итеративно:

```python
T = mat.tocsr()
while T.nnz != prev_nnz:
    prev_nnz = T.nnz
    T = (T + T @ T).astype(bool)
```

### 4.3. Тестовые данные

**`graphs/abab_cycle.csv`** - 6 вершин, 6 рёбер Цикл замкнут: финальное `b`-ребро ведёт обратно в вершину 0:

```
0,a,1
1,a,2
2,b,3
3,b,4
4,a,5
5,b,0   - замыкает цикл
```

**`graphs/abab_no_cycle.csv`** - 7 вершин, 6 рёбер. Финальное `b` - ребро ведёт в новую вершину 6, цикл не замкнут:

```
0,a,1
1,a,2
2,b,3
3,b,4
4,a,5
5,b,6   - нет замыкания
```

### 4.4. Результаты

#### Граф с циклом (`abab_cycle.csv`)

Алгоритм отработал все 4 фазы. Редукция источников на каждом шаге:

| Фаза | nnz матрицы | Источников | Граничные вершины |
|------|-------------|------------|-------------------|
| T_a  (фаза 1) | 4 | 6  | [1, 2, 5] |
| T_b1 (фаза 2) | 2 | 3 | [0, 3] |
| T_a2 (фаза 3) | 1 | 2 | [1] |
| T_b2 (фаза 4) | 0 | 1 | - |

Финальное произведение `T_a × T_b1 × T_a2 × T_b2` имеет `nnz = 0`, поэтому диагональ нулевая

Пофазный алгоритм проверяет прямую цепочку `a^m b^k a^i b^j` через конкретные граничные переходы. На данном графе `T_b2` оказалась пустой - второй блок `b` не смог найти ребро из вершины 1 (единственного оставшегося источника на фазе 4), ведущее обратно к стартовой вершине, так как ребро `5 -> 0` не начинается от вершины 1. Таким образом, цикл присутствует в графе, но пофазный алгоритм в данной конфигурации (с редукцией источников) его не обнаруживает - в отличие от стандартного подхода через полную грамматику, который рассматривает все пути через транзитивное замыкание.

#### Граф без цикла (`abab_no_cycle.csv`)

На фазе 3 (`T_a2`) множество граничных вершин оказалось пустым: из вершин `{3, 6}` (результат фазы 2) нет рёбер типа `a`. Алгоритм завершился **досрочно**:

```
Фаза 1 (T_a):  nnz=4, граничные вершины -> [1, 2, 5]
Фаза 2 (T_b1): nnz=2, граничные вершины -> [3, 6]
Фаза 3 (T_a2): nnz=0, граничные вершины -> []
Нет 2го a-блока -> циклов быть не может
```

### 4.5. Выводы по исследованию 2

Пофазный алгоритм успешно решает задачу поиска цикла для **произвольных значений `m, k, i, j`** без их явного перебора. Редукция множества источников на каждой фазе является практически значимой оптимизацией: в реальных разреженных графах большинство вершин отсекается уже на первых фазах, и последующие вычисления происходят на существенно меньших матрицах.

Выявлено отличие двух интерпретаций: **пофазный метод** ищет путь через конкретные граничные вершины, **стандартный подход** через полную грамматику ищет любой путь через полное транзитивное замыкание. Оба подхода корректны, но проверяют разные семантические условия
